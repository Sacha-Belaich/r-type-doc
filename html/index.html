R-Type Project Documentation

Overview

The R-Type project is a networked multiplayer game developed by our team of 5 developers: Deniz, Julien, Dimitri and Sacha. The game leverages modern C++ features and multiple powerful libraries to create an efficient and flexible architecture.

Technologies Used

We have integrated several tools and libraries to build R-Type:

  • SFML: For rendering graphics, handling window creation, and managing events.
  • Raylib: A lightweight library used as an alternative for specific graphical components.
  • Custom ASIO Wrapper: Our own wrapper around ASIO for asynchronous networking.
  • Custom ECS: A bespoke Entity Component System to manage game entities and their behaviors efficiently.
  • CPM (CMake Package Manager): For dependency management and integration of external libraries.
  • CMake: To configure, build, and compile the project.

Architecture

ECS (Entity Component System)

Our ECS implementation forms the core of the game engine, efficiently handling entities, components, and systems:

  • Entity: A unique identifier representing an object in the game world.
  • Component: Data structures that store properties for entities (e.g., position, velocity).
  • System: Logic that operates on entities with specific components (e.g., physics updates, rendering).

Example of entity creation and system logic:

ECS::Registry registry;
ECS::Entity player = registry.spawn_entity();
registry.add_system([](ECS::Registry &reg) {
std::cout << "Updating game logic for all entities." << std::endl;
});
registry.run_systems();

Architecture Overview

The game architecture follows a clean separation of concerns between the Client, Server, and ECS components:

  • Client: Handles rendering, user input, and communication with the server.
  • Server: Manages game state, processes inputs from clients, and sends updates back to clients.
  • ECS: Shared between the client and server to maintain consistent entity and system management.

Key interactions:

  1. The Server processes inputs and updates game logic using ECS.
  2. The Client renders the updated game state and sends player actions to the server.
  3. Networking via our Custom ASIO Wrapper ensures smooth communication between the client and server.

Networking

Networking is powered by our Custom ASIO Wrapper, which ensures smooth and asynchronous communication between the game server and clients:

  • Client-Server Architecture: A centralized server manages game state, while clients send inputs and receive updates.
  • Real-Time Updates: Optimized communication ensures minimal latency during gameplay.
  • Asynchronous Handling: All communication is handled asynchronously to avoid blocking operations and improve performance.

Building the Project

The project uses CMake with CPM for package management. Follow these steps to build the project:

  1. Clone the repository:
    git clone git@github.com:EpitechPromo2027/B-CPP-500-PAR-5-2-rtype-deniz.uzuntok-husson.git r-type
    cd r-type
  2. Configure and build:
    mkdir build && cd build
    cmake .. && cmake --build . --config Release && cmake --install .
    cd ..
  3. Run the server from root:
    ./r-type_server
  4. Run the client:
    ./r-type_client
